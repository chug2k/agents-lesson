The Access Agent in Mastra Studio

Last week's spike scripts ran outside Mastra's registry — nothing showed up in Studio. This week we port the agent in properly and watch it think.

1Port the agent into the registry

Four small files turn the spikes into a real Mastra project:

  • src/mastra/tools/who-is.ts — the fake-HR who_is tool.
  • src/mastra/tools/github-mcp.ts — GitHub remote MCP client; ~44 tools exposed, we keep 3.
  • src/mastra/agents/access-agent.ts — the agent: access policy in its instructions, HR + GitHub tools.
  • src/mastra/index.ts — new Mastra({ agents: { accessAgent } }). Registration is the whole trick.

2Boot Studio

cd week2 && npm install && npm run dev

Studio comes up at localhost:4111. The Access Agent is in the list because it's registered — that's the difference from the spikes.

3Open the agent

Note the tabs — Chat, Editor, Evaluate, Traces. Studio is a workbench, not just a chat window.

4Test it

Ask: "Priya is asking for read access to the repo chug2k/churn-dashboard. What do you recommend?"

The agent calls who_is (HR: employee, data team), then github_get_file_contents (the repo's README: maintained by the data team), joins the two against the policy — Verdict: GRANT
